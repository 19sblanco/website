Problem description

For those unfamiliar the traveling salesman problem is a famous problem in computer science that states given the location of a set of cities, find the shortest route that visits every city exactly once and returns to the starting city. Here is a more indepth description of traveling salesman problem the for those interested.

This problem is famous for being notoriously slow to solve with a computer because the only known way of solving it is to try out every possible path between the cities, meaning for N cities a computer would have to complete N factorial steps.

My process

First I just familiarized myself with the problem description itself. I read the first section of the wiki page but not more. I wanted to come up with a solution by myself. After thinking about it for a while I came up with my solution. I later found out that the solution that I came up with is a common algoirthm used in computer science called backtracking.

Backtracking is where all solutions for a problem is computed step by step recursively. You can read more about it here. It is ideal for problems where there is no solution more effiecient than trying out everything.
